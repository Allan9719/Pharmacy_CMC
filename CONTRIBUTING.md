# CMC 知识库贡献协作规则

## 一、贡献宗旨

1. 聚焦"中国医药 CMC 实操落地"，所有贡献内容需贴合本土法规 / 案例
2. 保证内容准确性：法规引用需标注来源，案例需隐去企业敏感信息
3. 协作共赢：欢迎医药研发 / 注册 / 质控 / 合规从业者共同完善

## 二、贡献范围

### 可贡献内容
- 本土 CMC 案例（发补 / 合规 / 失败案例）
- 可复用模板（申报 / 实操 / 合规类）
- 法规解读（新增 / 修订法规的 CMC 影响）
- 文档优化（错别字、逻辑梳理、标签补充）
- 前沿疗法 CMC 实操内容（ATMP / ADC / mRNA）

### 禁止贡献内容
- 无来源的虚假案例 / 数据
- 企业未公开的敏感信息
- 违反中国药品监管法规的内容
- 与 CMC 无关的广告 / 推广内容

## 三、贡献流程

### 分支管理规则
- **主分支（main）**：仅存放审核通过的稳定内容
- **开发分支（dev）**：用于日常内容迭代
- **案例分支（case-xxx）**：命名格式 `case-品类-年份`（如 `case-chemical-2025`）
- **模板分支（template-xxx）**：命名格式 `template-类型-名称`

### 提交流程
1. Fork 本仓库至个人账号
2. 创建专属分支（按上述命名规则）
3. 提交内容（需符合格式要求）
4. 发起 PR，标注"贡献类型 + 内容概述"
5. 核心维护者审核（1-3 个工作日），通过后合并

## 四、内容格式要求

### 案例类内容格式
```markdown
# 【品类】发补案例 - 问题类型（年份）
## 1. 案例背景
- 申报阶段：IND / NDA / 上市后变更
- 药品类型：化学药 / 生物药 / ATMP
- 申报企业类型：MAH / CDMO / 研发企业

## 2. 核心问题
- 法规依据：（中国药典 2025 版通则 / CDE 指导原则）
- 具体问题：

## 3. 整改方案
- 研究内容：
- 数据支撑：
- 法规适配：

## 4. 审评结论
- 整改后是否通过：
- 核心结论：

## 5. 避坑建议
- 实操要点：
```

### 法规解读类内容格式
```markdown
# 【法规名称】CMC 实操影响解读（发布时间）
## 1. 法规核心修订点
## 2. CMC 实操影响（申报/生产/合规）
## 3. 实操建议（短期/长期）
## 4. 参考资料（法规原文链接）
```

## 五、审核机制

1. 核心维护者：医药注册 / CMC 合规领域 5 年以上从业者
2. 审核维度：准确性、实用性、合规性
3. 审核反馈：未通过的 PR 需明确标注修改意见

## 六、版权与免责

1. 所有贡献内容的版权归贡献者 + 本仓库共同所有，允许非商业用途免费使用
2. 本仓库内容仅为实操参考，不构成法律 / 合规建议，具体申报需以 NMPA/CDE 官方要求为准
3. 贡献者需保证所提交内容无版权纠纷
