# 飞检/审计风险规避指南

> 聚焦中国药品监管飞行检查中的 CMC 高频缺陷与风险规避策略，帮助 MAH/生产企业提前防范。

---

## 1. 飞检概述

### 1.1 什么是飞行检查

飞行检查（Unannounced Inspection）是药品监管部门在**不预先通知**的情况下，对药品生产/研发企业进行的突击检查。检查重点是：

- 药品 GMP 合规性
- MAH 管控体系运行情况
- 数据完整性
- 偏差/CAPA 体系有效性

### 1.2 检查依据与机构

| 检查类型 | 执行机构 | 检查依据 |
|---------|---------|---------|
| 常规检查 | 省级药监局 | GMP（2020 修订版） |
| 飞行检查 | NMPA / 省级药监局 | 《药品生产监督管理办法》 |
| 有因检查 | NMPA | 基于投诉/不良反应/抽检不合格 |
| 注册核查 | CDE + 核查中心 | 《药品注册核查工作程序》 |

### 1.3 飞检常态化趋势

- 2024-2025 年飞检频次持续增加
- MAH 委托生产成为飞检重点方向
- 数据完整性检查力度加大
- 处罚力度趋严（警告、停产、吊证）

## 2. CMC 高频飞检缺陷

### 2.1 质量管理体系缺陷

| 缺陷类型 | 飞检发现 | 风险等级 |
|---------|---------|---------|
| 未建立质量管理体系 | MAH 仅有"名头"，无实际质量管理活动 | 高 |
| 组织架构不健全 | 未配备足够的 QA/QC 人员 | 高 |
| 未开展年度产品质量回顾 | APQR 流于形式或未执行 | 中 |
| 文件管理混乱 | 文件版本不受控、分发记录缺失 | 中 |
| 培训不足 | 关键岗位人员未接受 GMP 培训 | 中 |

### 2.2 生产与工艺缺陷

| 缺陷类型 | 飞检发现 | 风险等级 |
|---------|---------|---------|
| 未按批准工艺生产 | 实际生产工艺与注册批准不一致 | 高 |
| 工艺验证不充分 | 验证批次不足、参数范围未确认 | 高 |
| 设备校验缺失 | 关键设备未按期校验 | 中 |
| 清洁验证不完整 | 清洁验证未覆盖所有产品/设备 | 中 |
| 批记录不完整 | 批记录有涂改、缺失关键参数 | 高 |

### 2.3 实验室与数据缺陷

| 缺陷类型 | 飞检发现 | 风险等级 |
|---------|---------|---------|
| 数据完整性问题 | 删除数据、重复进样、修改日期 | 高 |
| OOS 未调查 | 超标结果未启动调查 | 高 |
| 分析方法未验证 | 检测方法未按 ICH Q2/药典 9101 验证 | 中 |
| 对照品管理不当 | 对照品超期使用、标定不规范 | 中 |
| 审计追踪缺失 | 色谱/天平等仪器未开启审计追踪 | 高 |

### 2.4 MAH 委托生产特有缺陷

| 缺陷类型 | 飞检发现 | 风险等级 |
|---------|---------|---------|
| 质量协议不完善 | 职责划分不清、变更管理缺失 | 高 |
| 未对受托方进行审计 | MAH 从未审计 CDMO | 高 |
| 未审核批记录 | MAH 未审核受托方批记录即放行 | 高 |
| 变更失控 | CDMO 变更未通知 MAH | 高 |
| 偏差管理缺失 | 偏差未报告 MAH、CAPA 未关闭 | 中 |

## 3. 飞检应对策略

### 3.1 日常准备（时刻准备被查）

```
飞检日常准备清单：
├── 文件管理
│   ├── 所有 SOP/批记录 版本受控、随时可调取
│   ├── 变更/偏差/CAPA 记录完整、可追溯
│   └── 供应商审计报告、质量协议 齐全
├── 人员
│   ├── 关键岗位人员熟悉自己的职责和 SOP
│   ├── 所有人员经过 GMP 培训并留有记录
│   └── 指定飞检接待负责人
├── 现场
│   ├── 生产区域符合 GMP 要求（洁净度、标识）
│   ├── 设备校验标签在有效期内
│   └── 偏离产品隔离存放、标识清晰
└── 电子数据
    ├── 色谱/天平/HPLC 审计追踪已开启
    ├── 电子记录备份完整
    └── 无共享账号、权限分级合理
```

### 3.2 飞检当天应对

```
飞检当天应对流程：
1. 确认检查人员身份和检查依据
2. 通知质量负责人和管理层
3. 安排接待区域和陪同人员
4. 提供检查所需的文件（如实、完整）
5. 对检查发现的问题逐条记录
6. 不隐瞒、不造假、不推诿
7. 检查结束后立即整理问题清单
```

### 3.3 整改报告撰写

```
整改报告框架：
1. 问题描述（准确引用检查发现）
2. 根因分析（Why-Why 分析 / 鱼骨图）
3. 纠正措施（已采取的即时措施）
4. 预防措施（防止再次发生的长期措施）
5. 完成时限与责任人
6. 有效性验证（整改后的验证数据）
```

## 4. 数据完整性专题

### 4.1 ALCOA+ 原则

| 原则 | 要求 | 常见违规 |
|------|------|---------|
| A — Attributable | 可归属（谁做的） | 共享账号、未签名 |
| L — Legible | 清晰可读 | 涂改、模糊不清 |
| C — Contemporaneous | 同步记录 | 事后补记 |
| O — Original | 原始记录 | 仅保存打印件，未保留电子原始数据 |
| A — Accurate | 准确 | 修改数据未说明原因 |
| + Complete | 完整 | 删除数据、选择性报告 |
| + Consistent | 一致 | 数据与记录矛盾 |
| + Enduring | 持久 | 记录保存条件不当 |
| + Available | 可获得 | 数据无法检索 |

### 4.2 电子数据管理要点

- 所有色谱仪、天平、pH 计等仪器必须开启审计追踪
- 禁止使用共享账号，每个操作员独立账号
- 电子数据定期备份（至少每日一次）
- 数据保留期限 ≥ 产品有效期 + 1 年
- 修改数据必须注明原因并经审批

## 5. 典型飞检案例

### 5.1 案例：MAH 未对受托方进行有效监督

```
检查发现：
- MAH 与 CDMO 签订了质量协议，但未约定审计频次
- MAH 从未派员驻厂或远程审核批记录
- CDMO 在未通知 MAH 的情况下更换了关键辅料供应商
- MAH 对上述变更不知情，已放行 3 批产品

整改要求：
- 补充对受托方的全面审计
- 评估变更对已放行产品质量的影响
- 修订质量协议，明确变更通知和审计要求
- 建立驻厂或远程监控机制
```

### 5.2 案例：数据完整性违规

```
检查发现：
- HPLC 审计追踪显示同一批样品多次进样，仅报告合格结果
- 操作员使用共享账号，无法追溯具体操作人
- 电子数据备份不完整，缺少 3 个月的原始数据

处罚结果：
- 警告信
- 暂停相关产品生产
- 整改通过后方可恢复生产
```
